## Current app state

The app is a functional prototype of a podcast client with:

- 4 screens: Home, Search, Queue, Library
- Mini player and full-screen player
- Navigation, favorites, downloads, reorderable queue (drag & drop)
- Persistence via `localStorage`
- **Real audio** ‚Äî HTML5 `<audio>` element with play/pause/seek/speed and auto-advance to next episode
- **Media Session (web only)** ‚Äî metadata, playback state and control handlers via `navigator.mediaSession`; works in desktop browsers but **not** inside the Android WebView (see below)
- **Real content** ‚Äî powered by the free **iTunes Search API** (no key, no signup, CORS-enabled):
  - Search returns real podcasts; podcast detail loads real episodes with real audio URLs
  - Home "Discover" shows the real Spanish top-podcasts chart
  - Real cover art everywhere (`Cover` renders the podcast image when available)
- **Real favorites & continue-listening persist** ‚Äî the metadata of favorited/played real podcasts and episodes is saved to `localStorage`, so they survive an app restart (not just their ids)
- **Demo seed data** ‚Äî a small set of hardcoded podcasts/episodes still seeds a fresh install until you follow/play real ones

## iTunes Search API

No credentials required. All CORS-enabled, so they work in the browser and the Android WebView:

- **Search podcasts:** `https://itunes.apple.com/search?media=podcast&term=<query>`
- **Episodes for a podcast:** `https://itunes.apple.com/lookup?id=<collectionId>&entity=podcastEpisode`
- **Top podcasts (Discover):** `https://itunes.apple.com/es/rss/toppodcasts/limit=<n>/json`

> Podcast Index was the original plan but blocks signups from free email providers (Gmail/Outlook); iTunes needs no account.
